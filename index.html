⚠ URGENT: API IS BROKEN ⚠

dev.ecomplier.com is misconfigured — the frontend cannot reach the backend!

WHAT HAPPENED:
Someone deployed a new server on Hostinger and changed the dev.ecomplier.com DNS A record to point to it — without configuring CORS or matching the backend API. The frontend at ecomplier.com now makes API calls that go to the wrong server, which blocks them with CORS errors. Every login attempt fails silently.
FIX #1 — DNS (GoDaddy):
Log into GoDaddy → DNS Management for ecomplier.com →

Update BOTH of these DNS records to point to the correct backend server:

1. @ (apex domain) — A record → [backend server IP]
2. www — A record → [backend server IP]

Also make sure dev.ecomplier.com points to the same backend server.
Remove any stale Hostinger IP records. TTL can stay at 600 seconds.
FIX #2 — CORS (Backend server config):
The backend must allow cross-origin requests from the frontend. Add BOTH domains to the allowed origins list: Access-Control-Allow-Origin: https://ecomplier.com Access-Control-Allow-Origin: https://www.ecomplier.com If using nginx, configure it in the server block: add_header Access-Control-Allow-Origin "https://ecomplier.com" always; add_header Access-Control-Allow-Origin "https://www.ecomplier.com" always; If using Express.js: app.use(cors({ origin: [ 'https://ecomplier.com', 'https://www.ecomplier.com', 'http://localhost:4200' ], credentials: true }));
FIX #3 — API ENDPOINTS:
The frontend POSTs to /Api/Admin/login with Content-Type: application/x-www-form-urlencoded and body username=...&password=....

The backend must return: { "status": true, "msg": "Loggedin.", "data": { "token": "<base64-encoded-token>", "adminid": 1, "first_name": "Admin", "last_name": "eComplier", "email_id": "admin@gmail.com", "mobile_no": "9890989000", "avtar": "", "gender": 1, "role_id": 0, "role_name": null, "userroleid": 0, "BankID": 0 } } All other endpoints follow the same pattern: POST to /Api/Admin/{action} with form-encoded body.
FIX #4 — DATABASE:
The current server returns "Invalid username or password" for valid credentials. Make sure the backend connects to the correct ecomplier_db database with the right SQL Server credentials. The original working backend has a working connection — use the same.
