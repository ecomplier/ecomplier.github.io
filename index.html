⚠ URGENT: API IS BROKEN ⚠

dev.ecomplier.com is misconfigured and blocking all frontend requests!

WHAT WENT WRONG:
Someone deployed a separate nginx/Node.js server on a Hostinger VPS (62.72.56.76) and pointed the dev.ecomplier.com DNS A record to it — without setting it up correctly.

That server has CORS locked to localhost:4200 only, which means the frontend at ecomplier.com cannot make ANY API calls. Every request fails silently.
HOW TO FIX — CORS (nginx config on the Hostinger server):
The nginx reverse proxy / Express server must allow cross-origin requests from the frontend domain. Access-Control-Allow-Origin: https://ecomplier.com Or use a wildcard for development: Access-Control-Allow-Origin: *
HOW TO FIX — API ENDPOINTS:
The frontend expects POST requests at /Api/Admin/login with Content-Type: application/x-www-form-urlencoded and body username=...&password=....

The backend must return JSON in this format: { "status": true, "msg": "Loggedin.", "data": { "token": "base64-encoded-token", "adminid": 1, "first_name": "Admin", "last_name": "eComplier", "email_id": "admin@gmail.com", "mobile_no": "9890989000", "avtar": "", "gender": 1, "role_id": 0, "role_name": null, "userroleid": 0, "BankID": 0 } }
HOW TO FIX — DATABASE:
The Hostinger server returns "Invalid username or password" for valid credentials. Make sure the Node.js backend connects to the correct SQL Server database (ecomplier_db) with the correct credentials. The original IIS backend has a working database connection — match it.
HOW TO FIX — DNS (if reverting):
If you want to stop using the Hostinger server, update the dev.ecomplier.com A record in the DNS provider back to the correct backend server IP.
